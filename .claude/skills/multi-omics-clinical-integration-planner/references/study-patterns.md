# Clinical–Multi-Omics Study Patterns

Use one dominant pattern and optionally one supporting pattern.

## Core patterns
1. **Clinical-risk augmentation pattern**
   - Goal: test whether omics adds value beyond clinical variables.
   - Output: baseline clinical model, augmented model, incremental value assessment.

2. **Subtype / patient-stratification pattern**
   - Goal: define clinically meaningful subgroups using aligned omics plus clinical context.
   - Output: subgroup logic, stability, and clinical anchoring.

3. **Outcome / response prediction pattern**
   - Goal: predict prognosis, treatment benefit, resistance, or severity.
   - Output: predictor design, validation path, calibration-aware performance framing.

4. **Mechanism-linked translational pattern**
   - Goal: connect clinical phenotype variation with cross-layer biology.
   - Output: biologically interpretable modules linked to clinical use.

5. **Biomarker prioritization pattern**
   - Goal: reduce multi-layer signal to a clinically usable biomarker panel or score.
   - Output: feature narrowing, interpretability, and validation ladder.

## Rule
Do not merge all five patterns into one omnibus design. Name the dominant pattern explicitly.
