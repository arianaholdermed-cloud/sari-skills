# Workload Configurations
# tumor-immune-infiltration-diagnostic-ml-research-planner

---

## Lite

| Attribute | Detail |
|---|---|
| **Goal** | Rapid proof-of-concept immune-linked diagnostic biomarker plan |
| **Timeline** | 2–4 weeks |
| **Data** | 1 discovery bulk cohort with disease vs control comparison; minimal metadata |
| **Core Modules** | DEG, one immune estimation route, one immune-linked candidate derivation route, one feature-selection route, ROC |
| **Validation** | Internal consistency only |
| **Figure complexity** | 4–5 figures |
| **Strengths** | Fast, executable, low barrier |
| **Weaknesses** | Weak generalizability, limited reviewer defensibility |
| **Typical target** | Pilot report, early feasibility work |

---

## Standard

| Attribute | Detail |
|---|---|
| **Goal** | Complete immune-linked diagnostic bioinformatics paper |
| **Timeline** | 1–2 months |
| **Data** | 1 merged or discovery bulk cohort + 1 independent validation cohort; immune inference-capable matrix; optional clinical variables |
| **Core Modules** | DEG, immune infiltration, WGCNA / module linkage or immune correlation, consensus ML compression, logistic model, nomogram, functional interpretation |
| **Validation** | External expression / diagnostic validation in one independent cohort |
| **Figure complexity** | 7–8 figures |
| **Strengths** | Conventional full-paper structure with interpretable logic |
| **Weaknesses** | Still vulnerable if validation cohorts are weak or cross-platform |
| **Typical target** | Standard diagnostic biomarker paper |

---

## Advanced

| Attribute | Detail |
|---|---|
| **Goal** | Stronger diagnostic manuscript with richer robustness and interpretability |
| **Timeline** | 2–3 months |
| **Data** | Discovery + validation + optional survival / protein / portal support |
| **Core Modules** | Standard modules plus stronger batch / robustness assessment, multi-tool immune support, calibration / decision curves, subgroup checks |
| **Validation** | Multi-layer validation across independent cohort(s) and support resources |
| **Figure complexity** | 8–10 figures |
| **Strengths** | Better reviewer-facing rigor and claim control |
| **Weaknesses** | More data dependency and more opportunity for inconsistency |
| **Typical target** | Better journals or stronger internal report |

---

## Publication+

| Attribute | Detail |
|---|---|
| **Goal** | High-ambition manuscript with stronger translational plausibility |
| **Timeline** | 3+ months |
| **Data** | Advanced-level data plus optional in-house tissue / protein / treatment-context support |
| **Core Modules** | Advanced modules plus stronger external portability checks and optional wet-lab follow-up |
| **Validation** | Cross-resource validation; optional tissue/protein confirmation |
| **Figure complexity** | 10+ figures / supplementary layers |
| **Strengths** | Strongest reviewer-facing package |
| **Weaknesses** | Higher workload and data / validation burden |
| **Typical target** | High-ambition submission or flagship internal project |
