# Reference Module 01: Question Fit and Design Entry

## Core Rule
A case-control design is appropriate only when the question can be structured around a clearly defined case state and a defensible control sampling logic from the same source population.

## Use This Module To Decide
- whether the user is asking an etiologic or exposure-association question,
- whether the outcome is rare enough or latency is long enough to justify case-control efficiency,
- whether a cohort design would be more natural,
- whether the user is actually asking a diagnostic, prognostic, or descriptive question instead.

## Appropriate Entry Patterns
Case-control design is often reasonable when:
- the outcome is rare,
- the endpoint has already occurred,
- prospective accrual would be slow or inefficient,
- exposure history can still be ascertained with acceptable credibility,
- incidence estimation is not the primary goal.

## Weak-Fit Patterns
Be cautious when:
- the user wants absolute risk or incidence,
- the user wants prognosis after a baseline state,
- the user wants longitudinal treatment response modeling,
- exposure timing is impossible to place before the outcome,
- case status is itself ambiguously defined.

## Important Distinction
Do not confuse:
- etiologic exposure research,
- diagnostic discrimination research,
- prognostic modeling,
- prevalence description.

These are not interchangeable just because all can be studied retrospectively.

## Reporting Rule
The output must explicitly state whether case-control design is:
- clearly appropriate,
- conditionally appropriate,
- weakly appropriate,
- or poorly aligned.
