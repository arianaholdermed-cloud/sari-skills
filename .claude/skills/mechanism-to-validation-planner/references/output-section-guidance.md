# Output Section Guidance

Use this module to keep the output decision-oriented and validation-sequenced.

## Section Priorities
- interpret the current finding clearly
- classify the finding type precisely
- state what the evidence supports and does not support
- map candidate validation layers
- recommend a primary route
- separate necessary from optional steps
- state whether clinical relevance should be included now
- identify the weakest link

## Style Rules
- keep sections short and explicit
- prefer stepwise structure when sequencing matters
- use tables only when comparison materially improves clarity
- do not hide uncertainty inside polished language
