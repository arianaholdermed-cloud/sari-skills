# Study Patterns — Detailed Logic
# conventional-non-oncology-hub-gene-research-planner

---

## Pattern A — Process-DEG Discovery Workflow

**Use when:** User wants disease DEGs intersected with a process-related gene family.

## Pattern B — Enrichment and GSEA Interpretation Workflow

**Use when:** User wants GO / KEGG / GSEA as a major interpretation layer.

## Pattern C — Hub-Gene Prioritization Workflow

**Use when:** User wants PPI-based key genes or central biomarkers.

## Pattern D — Regulatory-Network and Immune Interpretation Workflow

**Use when:** User wants TF/miRNA networks and immune infiltration.

## Pattern E — Multi-Layer Public Validation Workflow

**Use when:** User wants ROC support, expression validation, and coherent biomarker prioritization.

---

## Pattern Combination Rules

| Combination | When Appropriate |
|---|---|
| A + B | process-DEG discovery plus enrichment/GSEA interpretation |
| A + C | process-DEG discovery plus hub-gene prioritization |
| C + D | hub-gene study plus immune / regulatory interpretation |
| C + E | hub-gene study plus ROC and validation support |
| A + B + D | process discovery plus pathway interpretation plus immune-focused follow-up |
