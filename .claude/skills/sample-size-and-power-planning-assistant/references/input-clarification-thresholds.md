# Input Clarification Thresholds

## Core Rule
Do not produce a long sample-size planning answer unless the user has provided enough information to identify the primary design driver.

## Minimum Clarification Targets
Try to clarify the following before expanding:
- study type,
- primary endpoint,
- comparison structure,
- time horizon if relevant,
- main effect size target or clinically meaningful difference,
- event rate / prevalence / variance driver if relevant,
- available or feasible sample size.

## Ask Before Expanding
Ask follow-up questions first when any of the following are unclear:
- whether the study is hypothesis-testing vs pilot vs validation,
- what the true primary endpoint is,
- whether the user wants an exact formal estimate or a planning framework,
- whether the sample size is flexible or fixed by feasibility,
- whether survival/event-based logic is required.

## Question Style
Questions should shrink scope, not broaden it.
Prefer a short list of precise questions over an open-ended interview.
