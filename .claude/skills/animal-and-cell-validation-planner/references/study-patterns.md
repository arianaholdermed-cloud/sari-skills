# Validation Pattern Library

Choose one dominant pattern.

## 1. Expression-to-Phenotype Confirmation
Use when the user mainly needs to show that a candidate signal exists and tracks with a phenotype.
Typical sequence: characterization → perturbation or perturbation proxy → phenotype readout.

## 2. Causal Perturbation Validation
Use when the main question is whether changing the target/pathway changes the phenotype.
Typical sequence: baseline characterization → gain/loss perturbation → primary phenotype test → orthogonal confirmation.

## 3. Rescue / Specificity Validation
Use when specificity matters and a simple perturbation could be misleading.
Typical sequence: perturbation effect → rescue or reversal experiment → specificity-support readouts.

## 4. Mechanism-Chain Validation
Use when the user wants to test an intermediate pathway logic, not just endpoint change.
Typical sequence: perturbation → proximal mechanism readout → distal phenotype → optional rescue.

## 5. Drug-Response Validation
Use when the question centers on treatment sensitivity, resistance, or repositioning logic.
Typical sequence: baseline stratification → intervention → response readout → mechanism support.

## 6. Cell-First With Animal Escalation
Use when animal validation is potentially useful but not justified as the first move.
Typical sequence: cell proof-of-concept → decision gate → animal escalation.

## 7. Parallel Cell and Animal Validation
Use only when the user clearly needs two-context evidence and has feasible access.
Do not choose by default.
