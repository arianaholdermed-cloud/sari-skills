# Sprint Estimation — Detailed Rules

## Stage-Based Heuristics

| Stage | Sprint Range | Default Focus | Confidence |
|-------|-------------|---------------|------------|
| idea/outline | 12–18 | Contribution, framing, RQ, venue fit | Low |
| early draft | 8–14 | Theory, structure, method credibility | Medium |
| mature draft | 5–9 | Evidence, discussion, polish, compliance | High |
| revise-resubmit | 4–7 | Comment mapping, argument repair | Medium |
| rebuttal/camera-ready | 2–4 | Targeted fixes, traceability | High |

## Adjustment Factors

**Decrease the estimate if:**
- A strong existing contribution statement is present
- A clear theoretical framework is already in place
- Complete empirical results are available
- The author has prior successful submission experience at this venue

**Increase the estimate if:**
- Multiple co-authors need to be coordinated
- A novel methodology is involved
- Complex theoretical integration is required
- Significant reviewer concerns must be addressed

## Estimation Output Template

```markdown
**Sprint Estimate**: 8–12 sprints

**Reasoning**:
- Stage: Early draft (base: 8–14)
- Strong contribution claim (−2)
- Missing method details (+2)
- No empirical results yet (+2)

**Confidence**: Medium
**First Sprint Focus**: Contribution and theory framing
```

## Focus Shifts

### Default Progression

```
EARLY PHASE → MIDDLE PHASE → LATE PHASE → R&R PHASE
     ↓              ↓             ↓            ↓
Contribution    Rigor        Polish      Response
```

### Focus Shift Rules

1. **Explicitly declare every focus shift** — never drift silently
2. **Explain the reason** — state why the shift is occurring
3. **Log to the process log** — record the shift decision

### Blocking Situations

If a fatal block occurs:
1. Immediately stop the current focus
2. Shift to the highest-risk issue
3. Update the sprint narrative
4. Notify the user
