# Reproducibility and Reporting Rules

Use this module when assessing whether a method can be repeated, checked, or transferred.

## Audit Points
- software names and versions
- parameter transparency
- assay/protocol detail
- data/code availability
- workflow completeness
- cohort and preprocessing description
- threshold / cutoff clarity where relevant

## Important Rule
Absence of reporting is not proof of absence of rigor. Mark under-specification explicitly rather than inferring either adequacy or failure without caution.
