# Minimal Executable Version Template

When the original topic is too broad, define:

- **Core question:** the smallest defensible version of the question.
- **Phase 1 input:** minimum data / samples / access needed.
- **Phase 1 method chain:** shortest coherent analysis or experiment path.
- **Minimum validation:** least validation needed to avoid triviality.
- **First output:** pilot result, short paper seed, protocol seed, or decision memo.
- **Deferred components:** work that should move to phase 2 instead of blocking project launch.
