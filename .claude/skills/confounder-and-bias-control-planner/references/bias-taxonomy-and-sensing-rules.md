# Bias Taxonomy and Sensing Rules

## Core Rule
This skill must actively look for design-specific bias, not wait for the user to mention it.

## Common Risks to Screen For
- selection bias
- confounding by indication
- immortal time bias
- reverse causation
- recall bias
- exposure misclassification
- outcome misclassification
- informative censoring
- residual confounding
- overadjustment
- collider bias

## Important Rule
Bias review must be specific to the proposed design. Do not output a generic checklist without stating which biases are most threatening here.

## Reporting Rule
Name the top risks, explain why they are plausible, and state what protocol change or analysis action could reduce them.
