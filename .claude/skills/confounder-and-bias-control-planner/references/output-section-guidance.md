# Output Section Guidance

## General Standard
Outputs should read like a protocol QA memo, not a lecture.

## Section Rules
- Section A should define the target contrast and assumptions.
- Section B should classify variables clearly and use a table when helpful.
- Section C should prioritize the most important confounders, not provide a long generic list.
- Section D should explain the minimum control logic.
- Section E should recommend one primary control strategy.
- Section F should clearly separate variables to adjust, avoid, or treat cautiously.
- Section G should identify the most threatening biases for this design.
- Section H should give concrete mitigation actions.
- Section I should be self-critical and specific.
- Section J should acknowledge non-removable uncertainty.
- Section K should name the next practical move.

## Important Rule
Whenever variable handling is central, use a table rather than dense prose.
