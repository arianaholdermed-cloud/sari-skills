# Confirmatory vs Exploratory Rules

Use this module to classify each aim or analytic component correctly.

## Confirmatory Component

Use this label only when:
- the study question is pre-specified,
- the relationship or effect being tested is clearly defined,
- the evidence chain is adequate for that test,
- and the interpretation standard is stricter than general exploration.

## Exploratory Component

Use this label when:
- the work searches for patterns,
- multiple candidate features or associations may be screened,
- the result is better framed as hypothesis-generating,
- or validation is limited.

## Supportive Component

Use this label for analyses that help interpret the main result but do not independently carry the study.

## Reporting Rule

Do not use confirmatory language for discovery-heavy workflows, high-dimensional fishing, or thinly validated signals.
