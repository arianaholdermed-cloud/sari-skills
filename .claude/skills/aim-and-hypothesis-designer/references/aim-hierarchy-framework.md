# Aim Hierarchy Framework

Use this module to design a disciplined hierarchy of aims.

A study aim structure is not a topic list.
It should express one central study story, then subordinate all additional aims to that story.

## Core Aim Classes

### Primary Aim
Use this for the one aim that carries the study’s main scientific or clinical value.

A valid primary aim should:
- answer the central study question,
- define the main outcome or evidence target,
- support the future title-level framing of the study,
- and remain interpretable even if secondary aims are removed.

Do not assign multiple primary aims unless the user explicitly requests it and the logic remains coherent.

### Secondary Aim
Use this for aims that strengthen, extend, or contextualize the primary aim.

A valid secondary aim should:
- support the same study story,
- not require a separate study identity,
- and not displace the primary aim.

### Supporting Objective
Use this for embedded components that are useful but do not deserve full aim status.

Examples may include:
- comparator checks,
- subgroup analyses,
- mechanistic support elements,
- exploratory sensitivity layers.

## Important Rule

If one proposed “secondary aim” could become the title of a separate study, it may not belong in the same aim package.
