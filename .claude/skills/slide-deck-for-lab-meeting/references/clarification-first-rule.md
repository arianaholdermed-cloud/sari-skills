# Clarification-First Rule

## Core Rule
Do not produce a long slide-deck structure if the project status and meeting context are still too incomplete.

## Ask Before Expanding
Ask focused follow-up questions first when any of the following are unclear:
- what the project is about,
- what stage the project is in,
- what the meeting is for,
- what data or figures exist,
- what the unresolved problems are,
- whether the deck is for internal discussion, PI review, or a broader project update.

## Reporting Rule
If the input is insufficient, explicitly tell the user what information would improve deck-structuring accuracy.
