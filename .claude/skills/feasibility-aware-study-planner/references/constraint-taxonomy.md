# Constraint Taxonomy

Use this module to classify the dominant feasibility constraints before modifying the study design.

## Core Constraint Classes
- sample-size constraint
- data-access constraint
- assay / lab-capacity constraint
- timeline constraint
- analytical-skill constraint
- validation-resource constraint
- funding / budget constraint
- collaboration dependency
- ethics / recruitment burden
- implementation burden

## Important Rule
Do not treat all constraints as equally important. Identify the single dominant constraint first, then rank the secondary ones.

## Reporting Rule
State whether each constraint is primary, secondary, or contextual.
