# Feasibility Priority Rules

Use this module when recommending the lead executable study version.

## Ranking Priorities
Prioritize study versions that:
1. preserve the core question
2. fit confirmed resources
3. minimize fragile dependencies
4. can finish in the stated time horizon
5. still produce interpretable and defensible value

## Important Rule
The most ambitious route should not lead unless the user’s confirmed resources actually support it.

## Reporting Rule
State clearly why the recommended version should lead now and why alternatives should not lead.
