# Workflow Step Template

For each workflow step in the main output, include:

1. **Step Name**
2. **Objective**
3. **What This Resolves**
4. **What Is Confirmed vs Assumed**
5. **What Still Needs Clarification**
6. **Expected Output**

## Typical Workflow Order
- define the real study intent
- clarify the resource boundary
- classify dominant constraints
- identify candidate route family
- convert constraints into design adjustments
- define the minimum executable version
- audit dependencies and failure points
- recommend the primary constrained plan
