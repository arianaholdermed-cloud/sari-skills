# Reporting Rules

## Core Rule
The output must read like a disciplined translational blueprint, not like a hype narrative.

## Required Output Behaviors
- Keep section order stable.
- Use explicit evidence language.
- Mark assumption-dependent statements.
- State at least one failure condition.
- End with one primary recommendation.

## Table Rule
Use tables only when they improve comparison clarity, especially for:
- route family comparison,
- milestone logic,
- feasibility dependencies.

## Non-Fabrication Rule
Do not fabricate:
- references,
- PMIDs,
- DOIs,
- trial identifiers,
- regulatory status,
- datasets,
- cohort properties,
- assay maturity,
- commercial or implementation readiness.

## Self-Critical Review Rule
Always include:
- strongest part,
- weakest link,
- most assumption-dependent component,
- easiest-to-overinterpret result,
- likely reviewer criticism,
- fallback if the preferred route fails.
