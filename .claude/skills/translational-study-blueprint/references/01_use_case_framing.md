# Use Case Framing

## Core Rule
A translational blueprint must start by forcing one primary translational use case.

## Primary Use Case Families
- Diagnosis / detection
- Prognosis / risk prediction
- Treatment response prediction
- Patient stratification / subgroup assignment
- Therapeutic target / intervention development
- Monitoring / pharmacodynamic tracking

## Important Rule
These categories are not interchangeable.

A biomarker associated with disease severity is not automatically a diagnostic biomarker.
A prognostic signal is not automatically treatment-response predictive.
A biologically relevant target is not automatically development-ready.

## Reporting Rule
Always label:
- primary use case,
- secondary use case(s),
- why they should not be merged.

## Failure Pattern To Avoid
Do not write a blueprint that tries to serve diagnosis, prognosis, prediction, stratification, and therapy at the same time unless one route is clearly primary and the others are future expansions.
