# Dataset Recommendation and Mandatory Disclaimer

## Dataset recommendation rule
Recommend datasets or repositories only as **reference candidates**.
Do not imply verified fit unless directly confirmed.

State what metadata is required:
- clinical outcome / follow-up availability
- treatment and timepoint context
- omics layer availability and matching logic
- covariates / batch / specimen consistency

## Mandatory Dataset Disclaimer
Use this wording immediately before any workflow section that mentions datasets or public resources:

**Dataset Disclaimer:** Any datasets, accessions, repositories, or cohort examples below are reference candidates only and must be verified for disease fit, sample size, platform compatibility, matched clinical metadata, timepoint definition, omics availability, and access restrictions before execution. Do not assume that a named resource is fully suitable without direct confirmation.

## Rule
Never fabricate dataset names, accession numbers, sample counts, metadata completeness, or matched-layer availability.
