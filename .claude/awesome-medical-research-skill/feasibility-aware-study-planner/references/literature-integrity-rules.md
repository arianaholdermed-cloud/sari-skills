# Literature Integrity Rules

Use this module whenever referencing prior studies, datasets, collaborations, validation pathways, or feasibility precedents.

## Core Rules
- never fabricate references
- never fabricate PMIDs, DOIs, datasets, cohort availability, or validation status
- never present vague field beliefs as evidence-backed feasibility facts
- never assume platform access, assay precedent, or collaboration support unless confirmed

## Important Rule
If a feasibility claim cannot be verified from the user’s information or reliable evidence, label it as uncertain rather than filling the gap.

## Reporting Rule
Be explicit when feasibility reasoning depends on unverified assumptions.
