# Feasibility and Data-Quality Rules

Do not confuse desired protocol variables with confirmed availability.

Classify key data elements into:
- likely available
- potentially obtainable with extra effort
- currently uncertain or unavailable

Check explicitly:
- exposure ascertainment quality
- endpoint ascertainment quality
- follow-up completeness
- timing accuracy
- core covariate availability
- external validation realism

Important rule:
- If data quality assumptions are unverified, label the protocol as provisional in those areas.
