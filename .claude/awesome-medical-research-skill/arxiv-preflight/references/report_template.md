# Report Template

The report is a single Markdown file. The exact structure below is the contract — keep section headings, ordering, and the `Decision:` line stable so downstream tooling can parse them.

---

**Decision:** PASS | PASS_WITH_FIXES | HOLD

- Blockers: N
- High:     N

## Human Sign-off Checklist

Before submitting, the author should be able to check off:

- [ ] Every author has reviewed the final PDF/source.
- [ ] All references in this report verified or knowingly accepted.
- [ ] All tables and figures contain final data (no placeholders, no `XX%`).
- [ ] All `\ref{}` and `\cite{}` resolve.
- [ ] AI use, if significant, is disclosed in line with field norms.
- [ ] No AI tool appears in the author list or acknowledgments as a contributor of intellectual content.
- [ ] arXiv primary category is appropriate for the manuscript format.

---

## Decision rule

- **HOLD** — any `BLOCKER`. Submission is not ready.
- **PASS_WITH_FIXES** — no blocker, but at least one `HIGH` / `MEDIUM`, **or** reference verification is `INCOMPLETE`. When verification is incomplete, emit an explicit high-priority item telling the author to rerun citation checks with network access.
- **PASS** — only `LOW` items, no `INCOMPLETE` sections.

The decision line is the only thing many readers will glance at; keep it conservative. Do not bias toward `PASS` to be encouraging.

## Tone

- Direct, not apologetic. The author asked for a preflight; surface the issues.
- Locate everything. A finding without `file:line` or `page N` goes in "Recommended Cleanup", not "Blocking Issues".
- No editorialising about whether the paper is good. The skill checks readiness, not quality.
- Cite the source of policy claims (the arXiv pages in `arxiv_policy_notes.md`) when a finding turns on policy interpretation.
