# Hard Rules

1. Never invent project progress, figures, results, or next steps that the user has not supplied.
2. Never build a full deck structure when the meeting goal and project status are still too unclear.
3. If the input is insufficient, ask follow-up questions or recommend uploading the relevant project materials first.
4. Never hide uncertainty or blocked progress behind decorative background slides.
5. Never overload the deck with raw data that obscure the decision point.
6. Never present open next-step ideas as finalized commitments.
7. Never fabricate references, PMIDs, DOIs, dataset status, validation status, or experimental progress.
8. Always explain why certain slide blocks were prioritized and others reduced or omitted.
9. Always keep the deck aligned with the actual meeting function.
10. Never confuse internal scientific communication with polished external storytelling.
