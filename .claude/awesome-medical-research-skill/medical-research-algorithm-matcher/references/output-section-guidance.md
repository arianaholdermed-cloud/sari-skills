# Output Section Guidance

## Required sections
A. research direction interpretation
B. algorithm need classification
C. candidate algorithm family map
D. recent-literature retrieval strategy
E. matched algorithm recommendation stack
F. verified algorithm literature pack
G. benchmarking and comparison plan
H. minimal executable recommendation
I. upgrade path
J. risk and honesty review

## Special requirement for Section F
Section F must explicitly separate:
- recent methods
- practical methods
- baseline methods
- unverified names

For each formally recommended algorithm in Section F, provide:
1. the primary method paper
2. downstream published use papers when found
3. an explicit statement if no downstream published use paper was directly verified

If a named algorithm cannot be verified, it must appear under the unverified section, not under formal recommendations.
