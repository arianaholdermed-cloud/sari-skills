# Output Section Guidance
# result-reliability-checker

---

## Tone

The output should read like a **critical reliability audit memo**.
It should be direct, evidence-based, and usable for downstream grading or citation decisions.

---

## Writing Rules

- Prefer explicit reasons over soft impressions.
- Name the fragility source precisely.
- Separate what is reported from what is inferred.
- Separate result reliability from result importance.
- End with a clear usable judgment, not a vague “promising but needs more work.”
