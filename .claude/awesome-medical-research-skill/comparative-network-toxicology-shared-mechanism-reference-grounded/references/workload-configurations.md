## Lite

| Attribute | Detail |
|---|---|
| **Goal** | Rapid proof-of-concept comparative network-toxicology study |
| **Timeline** | 2–4 weeks |
| **Data** | Two exposure-target resources + one toxic-phenotype target resource |
| **Core Modules** | target retrieval, overlap decomposition, enrichment, one limited network or docking branch |
| **Validation** | discovery-level only or one lightweight plausibility branch |
| **Figure complexity** | 4–5 figures |
| **Strengths** | Fast, executable, preserves shared-vs-specific logic |
| **Weaknesses** | Limited orthogonal validation and mechanistic confidence |

## Standard

| Attribute | Detail |
|---|---|
| **Goal** | Complete comparative network-toxicology paper |
| **Timeline** | 1–2 months |
| **Data** | Lite data + PPI and docking resources |
| **Core Modules** | All Lite modules + PPI hub prioritization, docking support, structured comparative interpretation |
| **Validation** | Public or within-workflow orthogonal support |
| **Figure complexity** | 7–8 figures |
| **Strengths** | Meets typical reviewer expectation for comparative target-network toxicology papers |
| **Weaknesses** | Limited translational certainty |

## Advanced

| Attribute | Detail |
|---|---|
| **Goal** | Competitive multi-layer comparative toxicology paper |
| **Timeline** | 2–3 months |
| **Data** | Standard data + transcriptomic or AOP resources + stronger validation structure |
| **Core Modules** | All Standard + cross-check branch, richer shared-vs-specific synthesis, stronger reviewer-proofing |
| **Validation** | Multi-layer public support and richer interpretation control |
| **Figure complexity** | 8–10 figures |
| **Strengths** | Better reviewer-proofing and richer biological context |
| **Weaknesses** | More complexity; higher risk of overinterpretation if not controlled |

## Publication+

| Attribute | Detail |
|---|---|
| **Goal** | High-ambition manuscript with maximum reviewer defensibility |
| **Timeline** | 3–6 months |
| **Data** | Advanced data + stronger validation coherence + reviewer-facing downgrade logic |
| **Core Modules** | All Advanced + richer evidence layering, clearer claim-boundary map, stronger shared-vs-specific prioritization structure |
| **Validation** | Maximum within conventional public network-toxicology scope without overclaiming mechanism |
| **Figure complexity** | 10–12 figures |
| **Strengths** | Reviewer-proof structure; strongest paper logic in this family |
| **Weaknesses** | Major time investment; still not equivalent to mechanistic proof |

## Dependency Consistency Rules

### Core Principle
A downstream step may appear **only if its prerequisite data source, evidence layer, and method family have already been explicitly included** in that same configuration.

### Mandatory Dependency Rules

1. **Shared-vs-specific claims** may appear only when overlap decomposition is explicitly declared.
2. **Hub-target-dependent analyses** may appear only when PPI logic is explicitly included.
3. **Docking-dependent analyses** may appear only when receptor, ligand, and scoring rules are declared.
4. **Cross-check-dependent analyses** may appear only when transcriptomic or orthogonal support resources are declared.
5. **AOP synthesis** may appear only when the intermediate evidence layers are already present.
6. **Minimal Executable Version** must inherit only the modules declared in the Lite plan unless the skill explicitly states that this minimal version is being upgraded into a stronger variant.
