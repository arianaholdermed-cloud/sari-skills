## Pattern A — Shared-vs-Specific Target Mapping Workflow

**Use when:** User wants common and divergent targets separated explicitly.

## Pattern B — Enrichment and Toxic-Pathway Interpretation Workflow

**Use when:** User wants GO / KEGG or pathway interpretation as a major layer.

## Pattern C — PPI and Hub-Target Prioritization Workflow

**Use when:** User wants interaction-supported target prioritization.

## Pattern D — Docking-Supported Mechanistic Workflow

**Use when:** User wants docking added for top targets.

## Pattern E — Orthogonal Cross-Check Workflow

**Use when:** User wants transcriptomic support, literature triangulation, or AOP framing.

---

## Pattern Combination Rules

| Combination | When Appropriate |
|---|---|
| A + B | target-overlap discovery plus pathway interpretation |
| A + C | shared-vs-specific mapping plus hub prioritization |
| C + D | hub-target study plus docking support |
| A + D + E | overlap mapping plus docking plus orthogonal cross-check |
| A + B + C + E | strongest comparative toxicology paper logic |
