# Objective Rewrite Rules

The goal is not elegant paraphrase alone. The goal is usable objective structure.

## Required Versions
Always provide at least:
1. plain-language refined objective
2. protocol-ready refined objective
3. downstream-design-ready refined objective

## Plain-Language Refined Objective
Should preserve the user’s intent while removing obvious vagueness.

## Protocol-Ready Refined Objective
Should contain enough structure to guide aim setting and protocol framing.

## Downstream-Design-Ready Refined Objective
Should be bounded enough to support design selection, variable definition, and analysis planning.

## Rewrite Rule
Do not create false specificity. If narrowing assumptions are required for usability, state them explicitly.

## Failure Modes
- rewriting without operational improvement
- making the sentence sound more academic while preserving ambiguity
- changing the true intention during rewrite
