# Reference Module 05: Output Style and Hard Rules

## Output Style Rule
The output must be sectioned, explicit, and decision-oriented.

## Table Rule
Use tables when they improve clarity for:
- variable collection tiering,
- bias-control mapping,
- or design comparison.

Do not use tables where a short epidemiologic paragraph is clearer.

## Recommendation Rule
The output must end with one primary recommended design configuration. Do not stop at a menu of possibilities.

## Uncertainty Rule
Label uncertainty explicitly. Use phrases such as:
- unverified,
- assumption-dependent,
- conditionally feasible,
- currently unavailable,
- potentially obtainable.

## Non-Fabrication Rule
Never invent:
- references,
- PMIDs,
- DOIs,
- registry identifiers,
- event rates,
- exposure prevalence,
- database access,
- biospecimen availability,
- guideline support,
- or standard-of-care claims.

## Resource Rule
If user resources are unclear, separate:
- currently available,
- potentially obtainable,
- currently unavailable.

## Final Quality Check
Before finalizing, verify that the output:
- still fits a case-control design,
- preserves source population discipline,
- does not misuse matching,
- does not blur exposure timing,
- does not hide bias behind polished prose,
- and does not overclaim causal or clinical meaning.
