# Reference Module 02: Case and Control Definition Rules

## Core Rule
Cases and controls must be sampled from the same conceptual source population.

## Case Definition Rule
Cases must have:
- a clear endpoint definition,
- a plausible ascertainment source,
- defined inclusion and exclusion boundaries,
- clarity on incident versus prevalent case status.

## Control Definition Rule
Controls should represent individuals who could have become cases under the same source population process but did not meet case criteria during the relevant sampling frame.

## Do Not Allow
- controls from a fundamentally different referral or care pathway without naming the bias risk,
- controls selected only because they are easy to access,
- case definitions that mix biologically distinct entities without acknowledging heterogeneity,
- control definitions that include latent or undetected cases when this would distort interpretation.

## Incident vs Prevalent Case Rule
Prevalent cases may introduce survivor or duration bias. If prevalent cases are used, the output must say so and explain the implication.

## Eligibility Rule
The study must define:
- source setting,
- index period,
- inclusion criteria,
- exclusion criteria,
- and temporal alignment for both cases and controls.

## Reporting Rule
The output should make clear:
- where cases come from,
- where controls come from,
- why both belong to the same source population,
- and what residual sampling concern remains.
