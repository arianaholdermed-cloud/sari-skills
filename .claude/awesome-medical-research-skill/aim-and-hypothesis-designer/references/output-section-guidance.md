# Output Section Guidance

Formatting standard for Sections A–J:
- Keep each section explicitly labeled.
- Use compact, protocol-framing language rather than review-style prose.
- Section B should define the smallest coherent study story, not a broad motivation paragraph.
- Every aim entry should include: role, hypothesis status, required evidence type, and main dependency.
- Section E should make confirmatory vs exploratory status explicit.
- Section G should foreground failure modes, not bury them.
- Section H must recommend one final coherent aim package, not a basket of partially conflicting options.
- Section I must include self-critical review items even when the aim package looks strong.
