# Fallback Scenario Planning

## Core Rule
Always offer a fallback path when the ideal sample size is unlikely to be reached or when assumptions are fragile.

## Acceptable Fallbacks
- Use a smaller, more realistic effect size assumption
- Use a lower event-rate assumption
- Reframe to fixed-N feasibility evaluation
- Simplify to one primary endpoint
- Reduce subgroup ambitions
- Reduce predictor burden
- Separate pilot discovery from later validation
- Shift from confirmatory framing to signal-seeking framing
- Plan event-driven extension if feasible

## Important Rule
Fallback planning is not failure. It is part of responsible protocol design.

## Reporting Rule
State what should be simplified first if recruitment or event accrual underperforms.
