# Study Pattern Selection Rules

Choose one dominant pattern. If a second pattern helps, label it as a supporting layer.

## Common Dominant Patterns

### 1. Differential Biology to Pathway Interpretation
Best when the user wants to explain disease-control or subgroup molecular differences and derive pathway-level biological interpretation.

### 2. Clinical Association with Molecular Features
Best when the user wants to link omics signatures to outcome, severity, treatment response context, or quantitative clinical phenotypes.

### 3. Stratification and Subtype Discovery
Best when the user wants to identify molecular subgroups, latent classes, or risk strata with downstream clinical interpretation.

### 4. Multi-Omics Concordance and Priority Axis Identification
Best when transcriptomics, proteomics, metabolomics, or related layers are all plausible, but the key task is to determine which signal is shared, complementary, or dominant.

### 5. Biomarker Panel Development Support
Best when the user wants candidate-marker screening, prioritization, and staged validation rather than a purely descriptive omics report.

### 6. Mechanism-Supporting Translational Extension
Best when the goal is to support a mechanistic or translational hypothesis with bulk molecular evidence, not to prove causality by omics alone.

## Selection Rule
- Do not choose a more ambitious pattern than the metadata and sample structure can support.
- Do not default to subtype discovery if the user's real question is a defined contrast.
- Do not default to multi-omics integration if one layer is likely primary and the others are optional extensions.
