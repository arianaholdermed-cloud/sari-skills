# Feasibility and Constraints

## Core Rule
A good translational blueprint must expose where the route depends on resources that may not exist.

## Required Categories
Always distinguish:
- currently available resources,
- potentially obtainable resources,
- currently unavailable resources.

## Common Dependency Domains
- Cohorts and sample size
- Longitudinal follow-up
- Intervention exposure or treatment annotation
- Biospecimen type and quality
- Assay platform readiness
- Model systems for perturbation
- External validation datasets
- Clinical endpoint definition quality

## Reporting Rule
For each major dependency, state:
- why it matters,
- whether it is available,
- what fallback is possible if it fails.

## Failure Pattern To Avoid
Do not hide feasibility gaps behind generic wording like "future multicenter validation" if the actual dependency is currently unrealistic.
