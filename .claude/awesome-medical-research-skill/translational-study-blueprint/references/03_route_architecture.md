# Route Architecture

## Core Rule
Choose a route family that fits both the finding type and the intended translational endpoint.

## Common Route Families
- Biomarker -> assay -> retrospective validation -> performance evaluation -> implementation-oriented expansion
- Mechanism -> subgroup biology -> stratification logic -> retrospective clinical link -> prospective enrichment strategy
- Target -> perturbation evidence -> model-system efficacy -> translational biomarker support -> early development framing
- Signature -> reproducibility -> locked model / thresholding -> response-linked testing -> clinical-decision framing
- Phenotype -> measurable proxy -> longitudinal relevance -> monitoring or pharmacodynamic route

## Selection Rule
Choose the simplest evidence-aligned route that preserves the translational objective.

## Comparison Rule
When comparing routes, judge them by:
- biological fit,
- evidence fit,
- feasibility fit,
- dependence on unavailable resources,
- clarity of next milestone.

## Failure Pattern To Avoid
Do not select the most ambitious route simply because it sounds more translational.
