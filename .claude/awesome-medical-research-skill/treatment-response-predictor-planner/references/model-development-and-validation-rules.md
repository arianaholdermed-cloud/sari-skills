# Model Development and Validation Rules

## Core Rule
The protocol must state one lead prediction line and one realistic validation architecture.

## Must Specify
- modeling target
- model family or scoring logic
- thresholding or grouping concept if relevant
- internal validation plan
- external validation expectation
- transportability concerns
- whether recalibration or model updating may later be required

## Important Rule
Prefer the simplest model family that matches the data reality and intended deployment setting.
