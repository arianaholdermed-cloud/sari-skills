# Workload Configurations

Always output all four configurations. Recommend one as the primary plan.

| Configuration | Typical Scope | Data Expectation | Core Modules | Validation Level |
|---|---|---|---|---|
| Lite | Single trait, single molecular layer, bounded locus set | 1 suitable GWAS + 1 suitable QTL layer | Locus definition, harmonization, primary coloc, basic ranking | Internal consistency and transparent limits |
| Standard | Conventional publishable coloc study | 1 GWAS + 1–2 relevant QTL layers and clear tissue logic | Lite + stronger prioritization + annotation support + linked follow-up if justified | Cross-layer support at prioritized loci |
| Advanced | Stronger locus-resolution and robustness | Multiple QTL layers and/or stronger locus complexity handling | Standard + fine-mapping or conditional logic + deeper locus review | Multi-step robustness and ambiguity control |
| Publication+ | High-ambition multi-layer manuscript | Multiple QTL layers, strong locus coverage, and rich support resources | Advanced + cell-type/context extension and translational or mechanistic expansion | Multi-layer convergence with stronger follow-up |

## Recommendation Logic
- **Lite** = minimum credible locus-integration version.
- **Standard** = default best-fit unless the data are extremely limited or the ambition is unusually high.
- **Advanced** = use when locus complexity, reviewer expectation, or mechanistic depth materially justify more work.
- **Publication+** = use only when data completeness, tissue relevance, and extension support plausibly allow it.

## Subset Rule
Each higher configuration must extend the lower one. Do not introduce a completely different project at higher tiers.
