# Question-to-Route Matching Rules

Use this module to judge which route type actually matches the core research question.

## Core Rule
The primary plan must be selected by **question fit**, not by technical prestige, novelty, or personal preference.

## Match Cohort-Led Routes When
- the question is fundamentally clinical
- an outcome-linked human dataset is central
- population interpretation matters more than mechanistic perturbation

## Match Bioinformatics-Led Routes When
- the project is still discovery-stage
- signal generation is the dominant need
- existing datasets can answer the first-pass question reasonably well

## Match Mechanism-Led Routes When
- the central uncertainty is biological causality or function
- association alone would leave the main question unresolved

## Match Real-World Routes When
- practice heterogeneity, transportability, or implementation is central
- controlled or highly curated cohorts would miss the real decision problem

## Match Translational Validation Routes When
- the discovery already exists
- bridging toward practical usefulness is the main task

## Important Rule
Reject route options that sound attractive but do not answer the real question.
