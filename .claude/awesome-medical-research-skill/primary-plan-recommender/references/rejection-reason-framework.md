# Rejection Reason Framework

Use this module to explain why non-primary routes are weaker.

Common rejection or deprioritization reasons include:
- weak question fit
- premature for the current evidence stage
- dependency-heavy
- too indirect to answer the main question
- too resource-intensive relative to first-pass value
- better suited as validation or follow-up rather than lead route
- likely to produce attractive but strategically weak output

## Reporting Rule
Do not simply say an alternative route is "also possible." State why it should not be primary.
