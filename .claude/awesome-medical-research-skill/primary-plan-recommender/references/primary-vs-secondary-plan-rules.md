# Primary vs Secondary Plan Rules

Use this module to decide which route should lead and which routes should not.

## A route should be the primary plan when:
- it best matches the core question
- it can generate the most decision-relevant evidence first
- its dependency burden is acceptable
- its weaknesses are manageable rather than fatal

## A route should be secondary or follow-up when:
- it strengthens the primary route but should not lead
- it depends on prior signal confirmation
- it is best treated as mechanistic extension, validation, or downstream enrichment

## A route should be deferred or dropped when:
- it is structurally misaligned
- it is resource-heavy without answering the main question first
- it creates more complexity than decision value

## Rule
Always choose one leading route. Do not hide indecision inside a vague hybrid unless leadership logic is explicit.
