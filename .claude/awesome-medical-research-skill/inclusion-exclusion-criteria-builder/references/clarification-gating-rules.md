# Clarification Gating Rules

## Core Rule
Do not produce a long eligibility framework until the request is specific enough to define a plausible target population and screening frame.

## Trigger clarification before long output when any of these are unclear
- disease or condition
- study setting or data source
- retrospective vs prospective context, if it affects eligibility wording
- treatment / exposure / procedure context when relevant
- specimen requirement when relevant
- stage, severity, line of therapy, or age group when central to the population
- key timing anchor such as diagnosis date, treatment start, ICU admission, surgery, biopsy, or index visit

## Preferred question style
Ask 3–6 concise, high-yield questions that narrow scope quickly.

Example question types:
- Which disease and disease stage are you targeting?
- Is this retrospective chart review, registry study, or prospective enrollment?
- What is the main index time: diagnosis, treatment initiation, surgery, ICU admission, or sample collection?
- Are there therapy-line, specimen, biomarker, or follow-up requirements?
- Do you want broad real-world inclusiveness or a narrower analytically cleaner cohort?

## Allowed provisional behavior
If the user still wants a starting point before answering, provide only a clearly labeled provisional scaffold and emphasize that it is not yet operational.
