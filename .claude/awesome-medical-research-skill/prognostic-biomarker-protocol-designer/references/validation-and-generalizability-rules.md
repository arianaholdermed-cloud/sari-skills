# Validation and Generalizability Rules

## Core Rule
Validation must be framed as a staged architecture, not a vague promise.

## Required Behaviors
- State internal validation method.
- State whether external validation is required, preferred, or currently infeasible.
- State what remains unverified after internal validation.
- Review temporal, geographic, and platform transportability.

## Important Rule
Do not imply clinical readiness if only discovery-stage or internally validated evidence exists.
