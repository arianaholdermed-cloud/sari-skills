# Workload Configurations

Use four fixed configurations.

## Lite
Use when the user needs a feasibility-aware first-pass validation path.
- Usually one cell system or one minimal experimental context.
- Focus on baseline confirmation + one perturbation + one core phenotype/readout.
- Avoid unnecessary mechanistic layering.

## Standard
Use when the user needs a credible validation package suitable for an internal project, thesis chapter, or solid manuscript foundation.
- Usually one to two cell systems.
- Includes perturbation confirmation, phenotype testing, and at least one orthogonal readout.
- Animal work is conditional, not automatic.

## Advanced
Use when the user needs stronger mechanism or translational support.
- Often includes multiple cell contexts, rescue/specificity work, or conditional animal escalation.
- Should still remain claim-centered rather than maximal.

## Publication+
Use when the user explicitly wants a high-depth evidence ladder.
- Can include cell + animal + rescue/mechanism + translational-support extension.
- Only recommend when resource demands are realistic or explicitly sought.

## Primary Recommendation Rule
Always choose one configuration as the default primary plan.
Base the recommendation on:
1. fit to the claim,
2. feasibility,
3. minimum evidence needed,
4. risk of overbuilding.

Do not default to Publication+.
