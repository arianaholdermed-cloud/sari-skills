## Pattern Overview

This skill supports five FAERS study patterns. Pick one as the primary pattern, or combine them only when the dependencies are explicitly declared.

### Pattern A
Use when the paper is anchored on the main default story for this skill family.

### Pattern B
Use when the user wants a narrower clinically coherent safety subdomain or a focused characterization layer.

### Pattern C
Use when the user wants one extended dimension beyond the default pattern (for example within-class contrast or onset profiling).

### Pattern D
Use when the user wants a more explicitly reviewer-facing framing layer (for example curated PT panel or label-gap framing).

### Pattern E
Use when the user wants subgroup or translational expansion layered on top of the core design.

---

## Pattern Selection Rules

1. Pick the simplest pattern that matches the core question.
2. Do not combine multiple advanced layers unless the configuration explicitly supports them.
3. If one pattern depends on a data field not declared available, it must be removed.
4. The minimal executable plan should usually use only one primary pattern.
5. If a second pattern is added, state clearly whether it is **necessary**, **recommended**, or **optional**.

---

## Evidence Ceiling Reminder

No pattern in this skill family supports:
- incidence estimation
- absolute risk quantification
- definitive causal attribution
- prescribing or regulatory recommendations

All outputs must remain within pharmacovigilance signal and characterization boundaries.
