# Workflow Step Template
# medical-research-gap-to-study-planner

---

## Required 8-Field Format

Every step in the recommended protocol must use the following fields:

```
Step Name:        Short descriptive label
Purpose:          What gap component this step helps resolve
Input:            Exact data / materials / outputs required
Method:           Specific method(s) and why they fit the gap
Key Parameters /
Decision Rules:   Cutoffs, thresholds, inclusion logic, or acceptance criteria
Expected Output:  What success looks like and what file / result is produced
Failure Points:   What can go wrong; how to detect it
Alternative
Approaches:       Backup strategy if the primary route fails
```

---

## Typical Step Blocks

1. Gap-to-question refinement
2. Data or cohort selection
3. Endpoint harmonization
4. Core discovery analysis
5. Gap-specific resolution module
6. Validation module
7. Integrated interpretation
8. Deliverable and limitation synthesis

---

## Step Ordering Rules

- Clarify the question before selecting methods
- Fix endpoints and cohorts before downstream analyses
- Run the gap-specific resolution module before interpretation claims
- Validation must follow a clearly defined endpoint or mechanism
- Integrated claims must come after evidence-tier labeling

---

## Upgrade-Only Labeling

When a stronger publication-oriented module is added beyond the minimal executable plan, label it clearly:

```
[UPGRADE-ONLY]
Reason for Addition:
New Evidence Tier Enabled:
What It Strengthens:
```
