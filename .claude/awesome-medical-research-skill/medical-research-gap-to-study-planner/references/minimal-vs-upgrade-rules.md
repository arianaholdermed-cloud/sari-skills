# Minimal Executable Version vs Publication Upgrade
# medical-research-gap-to-study-planner

---

## Minimal Executable Version

The minimal executable version must be the **smallest credible protocol that still addresses the core gap**.
It should not be a weakened imitation of the full protocol.

Mandatory properties:
- directly answers the central gap
- uses only necessary modules
- keeps claims conservative
- avoids optional complexity

---

## Publication Upgrade Path

The publication upgrade version may add:
- stronger external validation
- orthogonal or protein support
- mechanistic or functional support
- cell-state or spatial context
- richer subgroup / response analysis

But every added module must state:
- why it is being added,
- what new evidence tier it enables,
- and how it strengthens the study beyond the minimal version.

---

## Forbidden Error

Do not strip the minimal version so far down that it no longer closes the core gap.
If the gap cannot be meaningfully addressed without the upgrade modules, then the minimal plan is not valid and must be redesigned.
