# Study Patterns — Detailed Logic
# non-tumor-mechanism-guided-diagnostic-ml-research-planner-aligned

---

## Pattern A — Mechanism-Guided Candidate-Restriction Workflow

**Use when:** User wants disease DEGs intersected with a mechanism-related gene family.

## Pattern B — Diagnostic-Model Construction Workflow

**Use when:** User wants explicit feature shrinkage and model building.

## Pattern C — Model Evaluation and Clinical-Utility Workflow

**Use when:** User wants ROC / calibration / DCA as a major evaluation layer.

## Pattern D — Regulatory-Network and Immune Interpretation Workflow

**Use when:** User wants TF/miRNA networks and immune infiltration.

## Pattern E — Multi-Layer Public Validation Workflow

**Use when:** User wants external validation, expression re-check, and coherent biomarker prioritization.

---

## Pattern Combination Rules

| Combination | When Appropriate |
|---|---|
| A + B | mechanism-guided candidate restriction plus model construction |
| B + C | model construction plus full evaluation |
| A + B + C | conventional mechanism-guided diagnostic paper |
| B + D | model-focused paper with immune / regulatory follow-up |
| B + C + E | strongest validation-oriented diagnostic-paper logic |
