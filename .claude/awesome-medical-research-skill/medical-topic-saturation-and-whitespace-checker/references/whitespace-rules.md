# Whitespace Rules

Use this module to identify meaningful remaining entry space.

Whitespace is not identical to “something has not been done yet.”
A valid whitespace angle should represent a meaningful remaining opportunity, not a cosmetic variation on a saturated template.

## Meaningful Whitespace May Include

- understudied but clinically meaningful populations
- stage-specific or setting-specific openings
- stronger endpoint framing than prior work
- better comparator design
- clearer validation path
- mechanism-to-translation bridge space
- assay or workflow improvements that materially change usefulness
- underdeveloped implementation layer despite repeated exploratory work
- orthogonal evidence layer still missing from the field

## Cosmetic Novelty Warning Signs

Do not call the following whitespace by default:
- changing only the algorithm name,
- minor feature-set variation,
- trivial subgroup slicing without real rationale,
- re-running the same template in a similar cohort,
- replacing one omics layer with another without changing the claim quality,
- renaming an already occupied problem.

## Reporting Rules

- Explain why a proposed whitespace angle is still open.
- Explain why it is not merely cosmetic novelty.
- State the likely burden required to convert the whitespace into a credible research entry.
